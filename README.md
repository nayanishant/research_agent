# Research Agent

A browser-based research assistant pipeline using LangChain + LangGraph states + scraping + Socket.IO streaming.

## Project structure

- `agent/`
  - `main.py`: state graph workflow (refactor query, search web, scrape, summarize).
  - `runner.py`: starts workflow and emits socket events.
- `server/`
  - `main.py`: FastAPI HTTP endpoints and socket routes.
  - `sockets.py`: global Socket.IO server instance.
- `searxng-docker/`: optional local web search engine setup (SearxNG with docker-compose).
- `requirements.txt`: Python dependencies.

## Setup (macOS/Linux)

1. Clone and activate venv:

```bash
cd research_agent
python3 -m venv venv
source venv/bin/activate
```

2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Create `.env` with API keys and optional settings:

```env
GOOGLE_API_KEY=<your_google_api_key>
OLLAMA_API_KEY=<if needed>
```

4. Run SearxNG (optional, for local search engine):

```bash
cd searxng-docker
docker compose up -d
```

Then ensure search API is available at `http://localhost:8080/search`.

## Run the app

1. Start the Socket.IO + FastAPI server:

```bash
uvicorn server.main:socket_app --reload --port 8000
```

2. Start the workflow (example using Python script or your client):

- HTTP POST to `http://localhost:8000/workflow/start`:

```json
{ "question": "what is RAG in LLM systems" }
```

3. Connect a Socket.IO client to stream updates:

- Connect to `http://localhost:8000`.
- Emit `join_room` with `{ "workflow_id": "<id>" }`.
- Listen for events: `room_joined`, `workflow_progress`, `workflow_done`, `workflow_error`.

## Local CLI (agent-only run)

If you want to run the `agent/main.py` directly as a CLI:

```bash
python agent/main.py
```

Then type your query when prompted.

## How the workflow works

1. The user query gets refactored into search queries by Gemini.
2. The app queries SearxNG (`/search?format=json&q=...`) and extracts top URLs.
3. Each URL is scraped (readability extraction) and text is truncated to keep tokens low.
4. The final summary is generated by Ollama and emitted back through Socket.IO.

## Troubleshooting

- If you get connection refused on 8080, ensure SearxNG is running and using the right URL.
- If your summary is empty, check scraping logs and search results.
- If API keys are invalid, set values in `.env` and restart.
